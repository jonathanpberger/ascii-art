# ADR 0001: ASCII Art Markdown File Constraints

- **Status:** Accepted
- **Date:** 2026-05-06
- **Authors:** Tutorial Author

## Context: Technical Requirements

We want agents to create ASCII art files for letter A-Z and numbers 1-100. We will
produce them in batches, refining our approach along the way. This file
includes technical requirements for the ASCII art we envision creating. 

This ADR pins down the **shape, size, and shell** of every ASCII art file so
that humans and agents both understand decisions we've made about constraints
and technical requirements. 

Aesthetic concerns (stroke style, case symmetry, visual balance) are not addressed
here. Visual design and content design standards are to be determined.

## Decision

Every ASCII art markdown file MUST satisfy the following rules.

### Location and naming

1. Files live in `ascii/` at the repository root. No subdirectories.
2. Each file is named `<character>.md`, where `<character>` is either:
   - a single lowercase letter `a`–`z` (e.g., `a.md`, `z.md`), or
   - a 1-to-3-digit decimal number from `1` to `100` (e.g., `1.md`, `42.md`,
     `100.md`).
3. No other filenames are permitted in `ascii/`.

### File structure

A conforming file contains exactly three elements, in this order:

1. **Exactly one H1 heading** whose text is the character itself, uppercased
   for letters and rendered as the bare digits for numbers. Examples:
   `# A`, `# Z`, `# 42`.
2. **Exactly one fenced code block** with the language identifier `text`
   holding the ASCII art.
3. **Exactly one rhyme**: a 2-line couplet of plain prose, immediately below
   the code block, with no surrounding heading.

No additional sections, headings, blockquotes, lists, images, links, HTML,
or trailing content are permitted.

### ASCII art constraints

Inside the fenced code block:

1. Maximum **40 lines** tall (line count, excluding the fence lines).
2. Maximum **40 columns** wide (longest line, measured in characters).
3. ASCII printable characters only, code points `0x20`–`0x7E`. No tabs, no
   non-ASCII, no trailing whitespace on any line.

### Rhyme constraints

1. Exactly **2 lines** of text.
2. Each line is a single sentence fragment or sentence; the two lines rhyme.
3. No markdown formatting (no bold, italics, links, or inline code).
